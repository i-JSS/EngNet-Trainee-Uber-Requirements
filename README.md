## Sobre o projeto

**2Getters** é um trabalho de engenharia de requisitos desenvolvido durante o processo trainee da **EngNet Consultoria**, empresa júnior de engenharia da UnB. A equipe atuou como uma consultoria de software, analisando o aplicativo **Uber** para compreender suas funcionalidades, fluxos e lógica de negócio — aplicando na prática técnicas reais de elicitação, modelagem e rastreabilidade de requisitos.

---

## Artefatos produzidos

### Pré-Rastreabilidade
Etapa inicial de entendimento do produto, antes da elicitação formal dos requisitos.

| Artefato | Descrição |
|----------|-----------|
| [É, Não é, Faz, Não faz](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/fazNaoFaz) | Delimitação do escopo do produto, definindo o que o Uber é e o que faz (e o que não é e não faz) |
| [Matriz SWOT](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/swot) | Análise estratégica do aplicativo mapeando forças, fraquezas, oportunidades e ameaças |

### Elicitação
Técnicas aplicadas para levantar os requisitos do sistema a partir de diferentes perspectivas.

| Artefato | Descrição |
|----------|-----------|
| [Análise de Protocolo](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/analiseProtocolo) | Verbalização em voz alta dos processos mentais do usuário durante o uso do aplicativo |
| [Personas](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/personas) | Perfis fictícios representando os principais tipos de usuários do Uber |
| [Caso de Uso](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/casoUso) | Modelagem das interações entre atores e o sistema em cenários específicos |

### Pós-Rastreabilidade
Mapeamento entre os requisitos levantados e os artefatos que os originaram.

| Artefato | Descrição |
|----------|-----------|
| [Forward-From](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/forwardFrom) | Rastreabilidade direta dos requisitos para os artefatos de implementação |
| [Matriz de Rastreabilidade](https://i-jss.github.io/requisitos-trainee-engnet/#/pages/finalizacao) | Visão consolidada da origem e cobertura de cada requisito |

## Contexto

Este projeto foi desenvolvido durante o **processo trainee da [EngNet Consultoria](https://github.com/EngNet-Consultoria)**, empresa júnior de Engenharia de Redes da Universidade de Brasília (UnB). O objetivo foi capacitar os membros nas práticas fundamentais de engenharia de requisitos utilizadas no mercado de desenvolvimento de software.
